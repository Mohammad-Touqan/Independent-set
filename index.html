The challenge is finding the Maximum Independent Set — the largest possible valid group. This is NP-complete: no fast algorithm is known for all graphs.

What is the Independent Set?

Read this before using the tool — understand the concept deeply

🌐 The Intuition

Think of a graph as a social network. Each node is a person. Each edge means two people know each other.


An Independent Set (IS) is a group where nobody knows anyone else in the group. No two chosen people share a friendship edge.

Formal definition: A subset S ⊆ V is an IS of G=(V,E) if for every u,v ∈ S, the edge (u,v) does NOT exist in E.

✅ Valid vs ❌ Invalid

Graph: 0 — 1 — 2 — 3 (a chain of 4 nodes)


✓
S = {0, 2} — Valid! No edge 0-2 exists.
✓
S = {1, 3} — Valid! No edge 1-3 exists.
✓
S = {0, 2, 3} — Valid! No edges 0-2, 0-3, or 2-3 exist. Size 3!
✗
S = {0, 1} — Invalid! Edge 0-1 exists.
✗
S = {1, 2} — Invalid! Edge 1-2 exists.
The IS problem asks: What is the largest valid IS? On P₄ the maximum IS has size 3.

🧩 How to Find an IS — Step by Step

1
Pick a node (greedy picks the one with fewest connections).
2
Add it to your IS. Mark it green.
3
Cross out all its neighbors. Mark them orange — they can't join.
4
Repeat with remaining (uncrossed) nodes.
5
Green nodes form your Independent Set.
Important: This greedy process doesn't always find the maximum IS. Guaranteeing the maximum requires exponential time — why IS is NP-complete!

🌍 Real-World Applications

📡
WiFi channels: Non-interfering access points form an IS — assign them the same frequency.
⚙
Register allocation: Non-conflicting variables form an IS and can share CPU registers.
📅
Scheduling: Non-conflicting tasks form an IS — they can run simultaneously.

🔑 Key Facts

Fact 1: {} (empty set) is always a valid IS. Size = 0.
Fact 2: Any single node {v} is always a valid IS. Size = 1.
Fact 3: S is an IS ↔ V\S is a Vertex Cover (complement relationship).
Fact 4: Finding the maximum IS is NP-complete — no known polynomial-time algorithm.
Fact 5: Checking if S is a valid IS is easy — check all pairs in O(|S|²). This is why IS ∈ NP.

NP-Completeness Proof

CLO-5 — Independent Set is NP-complete via reduction from Vertex Cover
We prove Independent Set is NP-complete in two steps: showing it is in NP, then reducing from the known NP-complete problem Vertex Cover. The key insight: IS and VC are complements.
🤔 What does NP-complete mean?▶

P — Problems solvable in polynomial time (fast).

NP — Problems where a solution can be verified in polynomial time.

NP-complete — Hardest problems in NP. Solving one solves all.

Like Sudoku: verifying a solution is easy (check rows/cols/boxes). Finding it is hard.

To prove X is NP-complete:
 ① X ∈ NP: a certificate can be verified in polynomial time.
 ② A known NP-complete problem Y reduces to X polynomially.

📋 Problem Definitions▶

Independent Set: Given G=(V,E) and k, does G have S⊆V with |S|≥k where no two in S are adjacent?

IS = { ⟨G,k⟩ | ∃S⊆V: |S|≥k ∧ ∀u,v∈S: (u,v)∉E }
Can you pick k or more nodes where no two are connected?

Vertex Cover: Given G=(V,E) and k, does G have C⊆V with |C|≤k such that every edge has ≥1 endpoint in C?

VC = { ⟨G,k⟩ | ∃C⊆V: |C|≤k ∧ ∀(u,v)∈E: u∈C ∨ v∈C }
Can you pick k or fewer nodes so every edge is "touched"?
Key complement: S is an IS ↔ V\S is a vertex cover. If S is independent, no edge has both endpoints in S — so every edge has ≥1 endpoint in V\S, covering all edges.
📐 Worked Examples — IS & VC on a Real Graph▶

Let's use this concrete graph: Triangle + extra node

Graph G
V={0,1,2,3}, E={(0,1),(0,2),(1,2),(2,3)}
✓ VALID Independent Sets
{0, 3} — No edge (0,3). Valid IS, size 2.
{1} — Any single node is always valid. Size 1.
{1, 3} — No edge (1,3). Valid IS, size 2.
{0, 3} is the Maximum IS — size 2 ⭐
✗ INVALID — edges exist inside the set
{0, 1} — Edge (0,1) exists! ✗
{0, 1, 2} — Edges (0,1),(0,2),(1,2) all exist! ✗
{1, 2} — Edge (1,2) exists! ✗
🔄 The Complement: Vertex Cover on the Same Graph
Since IS={0,3} is the max IS, the complement V\{0,3} = {1,2} is a minimum vertex cover.

Verify C={1,2} covers all edges:
• Edge (0,1): node 1 ∈ C ✓
• Edge (0,2): node 2 ∈ C ✓
• Edge (1,2): nodes 1,2 ∈ C ✓
• Edge (2,3): node 2 ∈ C ✓

All 4 edges are covered by just 2 nodes — this confirms the complement relationship!
💡 The Reduction in Action
VC question: "Does G have a vertex cover of size ≤ 2?"
↓ Apply reduction f: same graph, new target = |V| − 2 = 4 − 2 = 2
IS question: "Does G have an IS of size ≥ 2?"
Answer: YES — {0,3} is an IS of size 2 → so YES, a VC of size ≤ 2 exists ({1,2}).
The two questions have the same answer — this is the power of the reduction!

1
IS ∈ NP

Certificate: A subset S⊆V with |S|≥k.

Verifier — O(|V|²) time
1
Check |S| ≥ k. Cost: O(n).
2
For each pair (u,v) with u,v∈S: verify (u,v)∉E. Cost: O(n²).
3
Both pass → ACCEPT. Otherwise → REJECT.
Checking someone's answer is easy — just verify no two chosen nodes are connected. O(n²) is polynomial. IS ∈ NP. ✓

2
Vertex Cover ≤ₚ Independent Set

Reduction: f(⟨G,k⟩) = ⟨G, |V|−k⟩ (same graph, complementary target)

VC Input
Graph G=(V,E)
Cover size ≤ k
⟶f: same G,
target |V|−k
O(1) time
IS Output
Same graph G
IS size ≥ |V|−k
Don't change the graph — just flip the target number. Works because IS and VC are complements.

Forward (VC YES ⟹ IS YES):

If C covers all edges with |C|≤k, then S=V\C is an IS with |S|≥|V|−k.

Proof: If S weren't independent, ∃u,v∈S with edge (u,v). But u∉C and v∉C contradicts C covering that edge. ✓


Backward (IS YES ⟹ VC YES):

If S is an IS with |S|≥|V|−k, then C=V\S covers all edges with |C|≤k.

Proof: Any edge (u,v): since S is independent, not both in S, so ≥1 is in V\S=C. ✓


Polynomiality: f computes |V|−k and outputs the same graph — O(1) time. ✓

✓
Conclusion

✓
IS ∈ NP (polynomial verifier)
✓
Vertex Cover is NP-complete (Karp, 1972)
✓
VC ≤ₚ IS (polynomial reduction shown)

Therefore, Independent Set is NP-complete. ∎

Implication: No polynomial-time algorithm is known for maximum IS. We use greedy (approximate, fast) or brute force (exact, exponential).
🌍 Real-World Applications▶

Wireless Networks: Non-interfering stations form an IS — share the same channel.

Register Allocation: Variables that can share a register form an IS in the conflict graph.

Scheduling: Non-conflicting tasks form an IS — they can run simultaneously.

Reference: Cormen et al., Introduction to Algorithms, 3rd ed., Chapter 34.

🔗 Polynomial Reductions

The bridge between NP-complete problems — how we prove hardness
A reduction is a way of converting one problem into another. If we can reduce problem A into problem B, it means: "If you can solve B, you can also solve A." This lets us prove that Independent Set is hard by showing it is at least as hard as other known hard problems.
❓ What is a Reduction — and Why Do We Need It?▶

The big idea: A polynomial-time reduction from problem A to problem B is a function f that:

 ① Takes any input x for problem A
 ② Converts it into an input f(x) for problem B — in polynomial time
 ③ Guarantees: x is a YES-instance of A  ⇔  f(x) is a YES-instance of B
Think of it like a translation. If I can translate any French question into English, and you can answer English questions, then you can effectively answer French questions too — even if you don't speak French.

When do we use reductions?

To prove NP-completeness
Show that a known NP-complete problem (like Vertex Cover) reduces to your problem (IS). This proves your problem is at least as hard.
To solve a new problem
If you already have a solver for B, and you can reduce A → B, then you get a solver for A for free — just translate the input and use B's solver.
🔴 Reduction 1 (Project Proof): Vertex Cover →ₚ Independent Set▶

This is the reduction used in the final project to prove IS is NP-complete. It works because IS and VC are exact complements of each other on the same graph.

The Complement Relationship (the key insight):
S is an Independent Set of G  ⟺  V\S is a Vertex Cover of G.

Why? If S is independent, no edge has BOTH endpoints in S. So for every edge, at least one endpoint must be OUTSIDE S — meaning it's in V\S. That means V\S touches every edge = it's a vertex cover.
The Reduction Function f
Input: VC instance
Graph G = (V, E)
Target: cover of size ≤ k

Question: does G have a vertex cover of size ≤ k?
⟶
f(⟨G,k⟩) = ⟨G, |V|−k⟩
Same graph, flip target
O(1) time
Output: IS instance
Same graph G = (V, E)
Target: IS of size ≥ |V|−k

Question: does G have an IS of size ≥ |V|−k?
Step-by-Step Walkthrough on a Concrete Graph
The Graph G
Edges: (0,1),(0,2),(1,2),(2,3)
VC question: cover ≤ 2?
Apply f — Same graph, new k
|V| = 4 nodes
k = 2 (cover size)
New target = |V| − k
= 4 − 2 = 2

f(⟨G, 2⟩) = ⟨G, 2⟩
(same graph, same number here — coincidence that |V|−k = k)
IS question:
Does G have an IS of size ≥ 2?
Verify Same Answer
IS answer: YES
S = {0, 3} is an IS of size 2.
Check: edge (0,3)? NOT in E ✓
VC answer: YES
C = V\{0,3} = {1,2}
Check every edge:
(0,1)→1∈C✓ (0,2)→2∈C✓
(1,2)→1,2∈C✓ (2,3)→2∈C✓
Proof of Correctness — Both Directions
⟹ Forward: VC YES → IS YES If C is a vertex cover with |C|≤k, let S = V\C.
Then |S| = |V|−|C| ≥ |V|−k. ✓ size check

Is S independent? Suppose not — some edge (u,v) with u,v∈S. But u∉C and v∉C means C doesn't cover (u,v). Contradiction! So S is an IS. ✓
⟸ Backward: IS YES → VC YES If S is an IS with |S|≥|V|−k, let C = V\S.
Then |C| = |V|−|S| ≤ k. ✓ size check

Does C cover all edges? For any edge (u,v): since S is independent, u and v can't both be in S. So at least one is in V\S = C. ✓
Time to compute f: O(1) — we only compute |V|−k (one subtraction) and output the same graph unchanged. No new vertices or edges are created. Trivially polynomial.
🟣 Reduction 2 (Assignment): Clique →ₚ Independent Set▶

This is the alternate reduction required by Assignment 2. It uses a different technique: the complement graph instead of flipping the target number.

The Clique–IS Connection:
A set C is a Clique in G  ⟺  C is an Independent Set in Ḡ (the complement of G).

Why? A clique means every pair of vertices IS connected in G. In the complement, every pair is NOT connected. So a clique in G has no edges in Ḡ — that's exactly an independent set!
Input: Clique instance
Graph G = (V, E)
Target: clique size ≥ k

Question: does G have a clique ≥ k?
⟶
f(⟨G,k⟩) = ⟨Ḡ, k⟩
Complement graph
Same target k
O(n²) time
Output: IS instance
Complement graph Ḡ = (V, Ē)
Target: IS size ≥ k

Question: does Ḡ have an IS ≥ k?
How to Build the Complement Graph Ḡ
For every pair of vertices (u,v) where u ≠ v:
  • If edge (u,v) EXISTS in G → do NOT include it in Ḡ
  • If edge (u,v) does NOT exist in G → include it in Ḡ

This takes O(n²) time — iterate over all C(n,2) pairs and check each one.
Concrete Worked Example
Original Graph G
E={(0,1),(0,2),(1,3)}
Clique: {0,1} — both connected ✓
Build Ē (Complement Edges)
All 6 pairs on 4 nodes:
(0,1) in E? YES → NOT in Ē
(0,2) in E? YES → NOT in Ē
(0,3) in E? NO → ADD to Ē
(1,2) in E? NO → ADD to Ē
(1,3) in E? YES → NOT in Ē
(2,3) in E? NO → ADD to Ē

Ē = {(0,3),(1,2),(2,3)}
Complement Graph Ḡ
Green = IS {0,1}
No edge (0,1) in Ḡ ✓
Same answer: YES!
Proof of Correctness
⟹ Clique YES → IS YES Suppose C is a clique in G with |C|≥k.
Take any u,v ∈ C: edge (u,v)∈E (clique).
By complement: (u,v)∉Ē.
So C has no edges in Ḡ → C is an IS in Ḡ with |C|≥k. ✓
⟸ IS YES → Clique YES Suppose S is an IS in Ḡ with |S|≥k.
Take any u,v ∈ S: edge (u,v)∉Ē (IS).
By complement: (u,v)∈E.
So every pair in S is adjacent in G → S is a clique in G with |S|≥k. ✓
Time to compute f: O(n²) — iterate over all C(n,2) = n(n-1)/2 pairs and check each in O(1). This is polynomial. ✓

Why does this matter for the project?
Both reductions prove the same conclusion: Independent Set is NP-hard. Having two different reductions from two different problems (VC and Clique) makes the NP-hardness argument even stronger — IS is "reachable" from multiple directions in the NP-complete landscape.
📋 Quick Reference — Everything You Need to Know for the Q&A▶
3 things to say about any reduction: 1. What f does — describe the mapping
2. Correctness — prove ⟹ and ⟸
3. Polynomial time — give the O(·) bound
Key formulas to remember: VC→IS: f(⟨G,k⟩) = ⟨G, |V|−k⟩ — O(1)
Clique→IS: f(⟨G,k⟩) = ⟨Ḡ, k⟩ — O(n²)
Both are polynomial ✓
If asked "why does the reduction work?" Say: "Because IS and VC are complements — S is an IS if and only if V\S is a VC. So solving one immediately solves the other."
If asked "what is the conclusion?" Say: "Since VC (or Clique) is NP-complete AND VC ≤ₚ IS, IS is NP-hard. Combined with IS ∈ NP, Independent Set is NP-complete."

Algorithm Design — Greedy Paradigm

CLO-3 & CLO-4: Design justification, approximation analysis, counterexample, and worked example

1. Paradigm: Greedy Algorithm

The greedy algorithm makes the locally optimal choice at each step without reconsidering previous decisions.


Pseudocode
GREEDY-INDEP-SET(G = (V, E)):
  S ← ∅
  while V ≠ ∅:
    v ← node in V with minimum degree  // greedy choice
    S ← S ∪ {v}
    V ← V \ ({v} ∪ N(v))  // remove v and neighbors
  return S

Design intuition: A low-degree node has fewer connections. Adding it blocks the fewest future candidates, maximizing our chances of a large IS.

2. Worked Example — Cycle C₅

Graph: nodes {0,1,2,3,4}, edges {0-1,1-2,2-3,3-4,4-0}. All degree 2. Optimal IS size = 2.

Greedy Trace on C₅
1
All degree 2. Pick node 0 (tie broken by index). S = {0}.
2
Remove 0 and neighbors {1,4}. Remaining: {2,3}.
3
Remaining degrees: 2→d1, 3→d1. Pick node 2. S = {0,2}.
4
Remove 2 and neighbor {3}. Remaining: ∅. Done.
✓
Result: S = {0,2}, |S| = 2. Matches optimal! ✓

3. Counterexample — Star Graph K₁,₅ (Greedy ≠ Optimal)

Center node 0 connects to 5 leaves 1–5. Optimal IS = {1,2,3,4,5}, size 5.

Initial K₁,₅
Center d5, leaves d1
→
Greedy picks leaf 1
Picks leaf 1 (d1=min) → removes center 0 → all other leaves blocked!
→
Greedy: size 1 ✗
Only 20% of optimal!
Why did greedy fail? Picking a leaf removed the center, blocking ALL other leaves — even though {1,2,3,4,5} is a perfectly valid IS (no edges between leaves).
